Relações do Modelo Conceitual
USUARIO N:N ⟷ STARTUP Um usuário pode investir em várias startups e uma startup pode ter vários investidores → resolvido pela entidade HOLDING
USUARIO 1:N → ORDEM Um usuário pode criar várias ordens de compra ou venda
STARTUP 1:N → ORDEM Uma startup pode ter muitas ordens abertas no balcão
ORDEM N:N ⟷ ORDEM Uma ordem de compra pode ser casada com múltiplas ordens de venda → resolvido pela entidade TRANSACAO
STARTUP 1:N → PARCEIRO Uma startup possui vários sócios, mentores e conselheiros
USUARIO 1:N → PERGUNTA Um usuário pode enviar várias perguntas a startups diferentes
STARTUP 1:N → PERGUNTA Uma startup pode receber perguntas de vários usuários
USUARIO 1:N → MOV. CARTEIRA Cada movimentação de saldo pertence a um único usuário
STARTUP 1:N → HIST. PRECO Uma startup acumula um histórico de preços ao longo do tempo

3. Cenários Práticos — O Que Acontece no Banco
🎬 Estes cenários mostram exatamente quais documentos são criados ou alterados no Firestore quando uma ação acontece no app.
1

João se cadastra no app

Usuário preenche nome, e-mail, CPF, telefone e senha

WRITE
1

Firebase Auth cria o usuário

Gera o UID que será usado como ID do documento no Firestore.

2

Backend cria documento em users

✚ criado: users / usr_joao_001
"name": "João da Silva"
"email": "joao@email.com"
"cpf": "12345678900"
"walletBalance": 0.00 // começa zerado
"mfaEnabled": false
2

João adiciona R$ 1.500,00 de saldo fictício

Simulação de crédito interno — sem pagamento real

WRITE
1

Cria movimentação de crédito em wallet_txs

✚ criado: wallet_txs / wtx_cred_001
"userId": "usr_joao_001"
"type": "credit"
"amount": 1500.00
"balanceAfter": 1500.00
"description": "Recarga de saldo simulado"
2

Atualiza saldo em users

✎ atualizado: users / usr_joao_001
"walletBalance": 0.00 → 1500.00
3

João compra 10 tokens da AgroSense de Maria

Ordem de compra casa com ordem de venda aberta → transação executada

5 OPERAÇÕES
1

João cria ordem de compra

✚ criado: orders / ord_buy_001
"userId": "usr_joao_001"
"type": "buy"
"quantity": 10 · "pricePerToken": 12.50
"status": "open"
2

Backend casa com ordem de venda da Maria e cria transação

✚ criado: transactions / txn_001
"buyerId": "usr_joao_001" · "sellerId": "usr_maria_002"
"quantity": 10 · "pricePerToken": 12.50 · "totalValue": 125.00
3

Atualiza ordens e holdings

✎ atualizado: orders / ord_buy_001

✎ atualizado: orders / ord_sell_001
"quantityFilled": 0 → 10 · "status": "open" → "filled"

✚ criado/atualizado: holdings / usr_joao_001_stk_agrosense

✎ atualizado: holdings / usr_maria_002_stk_agrosense

Débita carteira do João e credita a da Maria
